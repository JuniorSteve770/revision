ÉTIQUETTES OBLIGATOIRES au début de chaque question :
[Terme → Définition]      — mémorisation d'un concept

[Ordre de dépendance]     — identifier les prérequis d'un concept

NIVEAU MOYEN ([[NOMBRE_MOYEN]] questions) :
- Focus : définitions, mécanismes de base, distinctions fondamentales
- Au moins 30% de questions [Confusion] ciblant les pièges les plus fréquents du sujet
- Options : 4 choix dont 1 correct, 1 piège technique plausible, 2 distracteurs réalistes
- Jamais d'options absurdes — toujours plausibles pour quelqu'un qui débute

NIVEAU AVANCÉ ([[NOMBRE_AVANCE]] questions) :
- Focus : patterns en contexte réel, anti-patterns, refactoring, multi-concepts
- Questions basées sur du code réel ou des scénarios architecturaux
- Au moins 2 questions [Anti-pattern] ou [Refactoring]
- Au moins 1 question [Code → Analyse]

NIVEAU EXPERT ([[NOMBRE_EXPERT]] questions) :
- Focus : architecture complète, raisonnement système, cas ambigus, ordre de dépendance
- Questions ouvertes sur des choix de conception avec trade-offs explicites
- Au moins 1 question [Nommage inversé]
- Au moins 1 question [Ordre de dépendance]
- Les distracteurs experts sont aussi techniques que la bonne réponse

RÈGLES COMMUNES :
• Chaque explication (explanation) contient : règle centrale + contre-exemple ou "pourquoi ça échoue" + ancrage dans [[CONTEXTE_METIER]]
• Jamais plus de 4 phrases dans explanation
• La bonne réponse ne doit pas être évidente à la longueur (éviter "la réponse la plus longue est correcte")
• Les questions progressent dans la logique du cours — les prérequis testés avant les concepts avancés

LOGIQUE REACT À REPRODUIRE EXACTEMENT
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

useState : level ("basic"|"moyen"|"avance"|"expert"), currentSlide, currentQuestion,
           scores {moyen:0, avance:0, expert:0}, timeLeft (25), showResult, message

useEffect #1 : timer 25s par question (level !== "basic" && !showResult)
               → decremente timeLeft → appelle handleNextQuestion à 0

useEffect #2 : auto-avancement slides toutes les 12s (level === "basic")
               → setCurrentSlide → quand fini, passe à level "moyen"

handleNextQuestion : avance la question → si fin de niveau, passe au suivant
                     (moyen → avance → expert → showResult=true)

handleAnswerClick : compare option à current.answer → met à jour scores
                    → setMessage (✅ ou ❌ + explanation) → setTimeout(handleNext, 4000)

renderInlineTokens : parse **bold**, `code`, *italic* en JSX inline
renderFormattedText : split sur " ◆ " → affiche chaque segment avec ◆ bleu devant
